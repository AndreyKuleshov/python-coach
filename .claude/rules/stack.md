# Tech stack — single source of truth

Treat this list as canon. Adding, removing, or replacing any of these without explicit user approval breaks the contract — escalate via `AskUserQuestion` instead of adopting unilaterally. The point is to keep the build from drifting into a zoo of half-overlapping tooling.

## Base

| Layer | Technology |
|---|---|
| Language | **Python 3.13+** (latest stable) |
| Package manager | **uv** (+ `uv workspaces` if the repo grows past one member) |
| Async runtime | **asyncio** (stdlib). **No `uvloop`** |
| ASGI server | **uvicorn** |
| Linter / formatter | **ruff** + **ruff format** |
| Type checker | **pyright** — only one. **No mypy / basedpyright / pyrefly** |
| Architecture enforcement | **import-linter** — encodes the layer dependency direction from `api-layers.md` |

## HTTP / REST

| Layer | Technology |
|---|---|
| Framework | **FastAPI** |
| Validation / DTOs | **Pydantic v2** (+ **pydantic-settings** for config) |

Auth is out of scope for the MVP (single-user platform). If added later, keep it custom and minimal — do not pull in a framework unilaterally.

## Database

| Layer | Technology |
|---|---|
| ORM (declarative) | **SQLModel** (on top of SQLAlchemy 2.0+) — tables double as domain models (see `api-layers.md`) |
| Lower-level ORM | **SQLAlchemy 2.0+** (async, typed) — used directly for joins / hybrid properties / hand-tuned queries |
| Driver | **asyncpg** (Postgres) |
| Migrations | **Alembic** — described in Python via `op.create_table(...)`, autogenerated from SQLModel where possible. No SQL dumps in the repo |

## Code-execution sandbox (the platform's core risk)

The platform runs **arbitrary user code**, so the runner is isolated, not in-process. Canonical approach: a **Docker** container per submission with CPU / memory / wall-time limits, **no network**, and a non-persistent filesystem. The architect's threat model + isolation mechanism is the authority here; do not run user code in the app process. No alternative sandbox tech (gVisor, firejail, nsjail, etc.) without explicit approval.

## Observability

- **structlog** for structured JSON logs. No `print` in services (see `all-languages.md`).

## Tests

| Layer | Technology |
|---|---|
| Runner | **pytest** + **pytest-asyncio** |
| API tests | **httpx** (`AsyncClient` + `ASGITransport`) |
| UI tests | **Playwright** + **pytest-playwright** |
| Reporting | **Allure** (optional) |

See the `qa` agent and `.claude/references/qa/` for authoring conventions.

## Optional / add only when needed (with approval)

- **Redis** (`redis.asyncio`) — only if a job queue or rate-limit for the sandbox is actually needed. Not adopted by default.

## When the user proposes something off-list

Stop. Surface the gap, summarise the trade-off, and ask whether to extend this canonical stack — never adopt unilaterally.
